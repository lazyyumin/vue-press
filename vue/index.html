# vue 响应式原理(MVVM模式)

vue响应式原理是数据劫持结合观察者模式 在vue2中，把一个普通对象传入vue实例中的data时，vue会遍历对象的属性，使用defineProperty把对象的属性使用setter和getter进行劫持
每个响应式属性都会创建一个Dep对象，相当于发布者，负责收集依赖添加观察者(watcher)，当响应式对象更新时触发setter，Dep会触发notify通知watcher使用update进行更新
vue3中，使用的proxy进行数据劫持

# vue diff

# vue2使用双端diff

  • 全端diff
  • 通过新前与旧前、新后与旧后、新后与旧前、新前与旧后、暴力比对5种查找方式
  • 暴力对比，及前4中都不适用，会对旧的节点进行循环，生成一个映射表，使用新节点的key去映射表中查找，找到了就把该节点移动到最前面，原来的位置用undefined占位，映射表中没有就插入新节点
  • 循环结束后如果新节点有剩余，循环列表逐个插入；旧节点有剩余，直接删除

# vue3使用快速diff

  • 在模板编译阶段添加一个patchFlag，如果patchFlag小于0，不需要diff，大于0才需要diff
  • 处理相同前置节点和相同后置节点
  • 新节点有剩余，插入
  • 老节点有剩余，删除
  • 都有剩余
    1. 构建一个source数组
    2. 遍历新节点，构建映射表
    3. 遍历老节点，在映射表中查找，找到就在source数组中记录老节点的位置，没找到就是-1
    4. 通过source数组计算出最长递增子序列来得到出不需要移动的节点，再将新节点的剩余节点插入到正确位置

# vue中key的作用

key的作用是在diff中判断两个节点是否是相同的唯一标识

# 为什么不推荐index作为key?

  • 使用index作为key时，当我们进行在数据前面插入一条数据时，index顺序就会发生变化，导致旧节点不能复用，需要重新生成新节点，造成不必要的性能消耗

  • 删除第一项时，导致最后一项被错误删除

# nextTick实现原理

  • vue修改数据后不会立即更新dom，而是等待数据更新完毕之后再统一更新视图
  • nextTick的作用就是可以将里面的回调函数延迟下次 DOM 更新循环结束之后执行。在修改数据之后立即使用这个方法，获取更新后的 DOM。

# vue3中ref和reactive的区别

  • ref 一般用来声明基本数据类型的变量
  • reactive 声明引用数据类型的变量

# reactive赋值响应式会丢失，但ref不会的问题

  • reactive 定义对象时是直接调用 proxy 进行处理，重新赋值会丢失原来的响应式对象引用地址，变成新的引用地址，新对象是没有经过 reactive 处理的普通对象，所以会丢失响应式
  • ref 定义对象时，会先转换为 RefImpl 对象，无论是修改还是赋值都会调用 setter ，使用 reactive 处理为响应式对象，所以不会丢失响应式。

# vue2中sync修饰符的作用和用法

sync是一个语法糖，是父组件监听子组件更新某个props的缩写 vue的组件不能更新props，只能通过父组件的事件来更新 在父组件中绑定的元素上使用sync修饰符，子组件直接使用update:name来进行更新

//  父组件中
<Child :propsName.sync="name"/>

//  等价
<Child :propsName="name" @update:propsName="name=$event"/>

//  子组件
<script>
  export default {
    props: [propsName]
    methods:{
      function (){
        this.$emit("update:propsName", newData)
      }
    }
  }
</script>
